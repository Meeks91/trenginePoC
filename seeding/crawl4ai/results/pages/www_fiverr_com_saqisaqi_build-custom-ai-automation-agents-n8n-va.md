Can you build RAG (document Q&A)? What formats do you accept?
Yes — I ingest PDFs, DOCX, TXT, CSV, and common cloud links. For best results, give clean source files (organized folders). Large corp datasets (>10k documents) require a pre-order discussion and may change price/time.
Which voice/calling platforms do you support and who pays for phone/API costs?
I work with VAPI and Twilio (and can integrate other providers). You must supply or pay for phone numbers, provider accounts and API usage. I’ll advise on cost-effective setup and configure it for you.
Which CRMs and tools can you integrate?
HubSpot, Salesforce, Zoho, Google Sheets, Airtable, n8n, Zapier/Make, and custom REST APIs. Provide API keys and required permissions. Complex or admin-level integrations may need extra time/fee.
Will I get source code, docs and deployment files?
Source code, setup files, and detailed comments are included only in Enterprise or as paid extras. I always deliver a README and testing logs; choose extras (source code/comments/setup) if you need a full handover.
How do you test the agent and what do live-call tests require?
I run staged tests (local → dev → live). For live-call tests, I need test phone numbers and confirmation that the numbers consent to be called. Live testing may incur provider call costs (paid by the buyer) and is available as an extra.
Are there ongoing or third-party costs?
Yes — hosting, OpenAI/GPT tokens, vector DBs, VAPI/Twilio call minutes, phone numbers, and any paid APIs are paid by you. I can recommend options and include a monitoring/maintenance plan as an add-on.
Can you handle sensitive or regulated data (NDA / HIPAA / GDPR)?
I can sign an NDA. For regulated data (PHI/PCI), we must discuss secure hosting, encryption, and compliance requirements before ordering — these generally require custom quotes and extra fees.
When should I contact you before ordering?
Please message me first for: enterprise projects, >10k documents, multi-language agents, legal/compliance-sensitive systems, custom LLM training, onboarding >3 third-party integrations, or timelines shorter than rush extras. Contacting first avoids delays and misquotes.
